# VZ Price Guide --- Product Map

*Last updated: 2026-02-25*

------------------------------------------------------------------------

## 1. What VZ Price Guide Is Today

VZ Price Guide is a search-driven Minecraft economy utility.

Users primarily arrive via organic search to: - Look up item prices -
Export price lists - Quickly validate economy ratios

The product currently functions as a high-quality episodic tool rather
than a recurring platform.

------------------------------------------------------------------------

## 2. Core Strengths

### Strong Organic Traffic

-   \~50% of sessions from Organic Search
-   High engagement quality (57% engagement rate from Organic)

### High Engagement Among Power Users

-   \~20 searches per searcher
-   2+ minutes average engagement time
-   Deep interaction on core price lookup page

### Strong Signup Conversion

-   \~12% visitor → account conversion
-   \~45% of new signups complete export

These are strong early-stage signals.

------------------------------------------------------------------------

## 3. Core Weaknesses

### Low Retention

-   \~17% returning users
-   Sharp drop after Day 1
-   DAU/MAU ≈ 4%

### Single-Feature Perception

-   Majority of traffic interacts only with price lookup
-   Shop Manager and extended tools have low usage

The product currently lacks a natural return loop.

------------------------------------------------------------------------

## 4. Current Product Identity

VZ Price Guide is currently:

> A high-quality search utility.

It is NOT yet: - A sticky SaaS product - A recurring ecosystem - A
community platform

This is not a flaw --- it defines the growth strategy.

------------------------------------------------------------------------

## 5. Near-Term Strategic Goal

Increase retention from \~17% → 25%.

Focus: Introduce lightweight return mechanics that extend natural lookup
behaviour.

Avoid: Building large, high-friction systems (e.g., complex shop
management flows) without proven demand.

------------------------------------------------------------------------

## 6. Retention Experiments (Aligned With Current Behaviour)

### 1. Saved Custom Price Guide

Allow users to: - Add searched items to a custom list - Save and revisit
later

Goal: Convert passive lookup → lightweight ownership.

------------------------------------------------------------------------

### 2. Server Preset Templates

Provide starter economy presets: - Survival preset - Diamond economy
preset - Hardcore preset

Goal: Give users identity and a reason to refine over time.

------------------------------------------------------------------------

### 3. Economy Balance Calculator

Allow users to input core item prices (e.g., diamond, iron, emerald) and
receive: - Ratio analysis - Inflation warnings - Suggested corrections

Goal: Create analytical return behaviour.

------------------------------------------------------------------------

### 4. Recently Viewed Items

Save and display recent item history for logged-in users.

Goal: Increase frictionless return usage.

------------------------------------------------------------------------

## 7. Growth Model Decision (Long-Term Fork)

### Path A: Optimised SEO Utility

-   Scale traffic
-   Improve export UX
-   Monetise via volume

### Path B: Sticky Ecosystem

-   Saved state
-   Personalisation
-   Tools & analytics
-   Recurring user value

This decision should be informed by retention experiment results.

------------------------------------------------------------------------

## 8. 30-Day Focus

1.  Implement one lightweight retention mechanic.
2.  Measure returning user % after 30 days.
3.  Avoid expanding feature surface prematurely.
4.  Continue strengthening SEO authority.

------------------------------------------------------------------------

## 9. Guiding Principles

-   Build extensions of existing behaviour, not new paradigms.
-   Minimise setup friction.
-   Prioritise return loops over feature breadth.
-   Measure before expanding scope.

------------------------------------------------------------------------
